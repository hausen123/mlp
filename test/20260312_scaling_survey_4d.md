# スケーリングサーベイ（4d FFN） — 独立性クエリ品質 vs データ量

## 目的
FFN中間層を4dに拡張したMLPMemoryで、前回（2d FFN）と同様のスケーリング実験を行い
品質劣化閾値を比較する。

## 変更点
- FFN: Linear(d, 2d) → Linear(d, 4d)（mlp.py MLPBlock）

## データ準備
- 元JSONL: `data/202510301810_250304_kangaekata_rag_workflow.jsonl`（273,516件）
- `data/wo_independence.jsonl`: 元JSOLから「原子力規制委員会」AND「独立性」を含む行を除外（272,139件）
- `data/independence.jsonl`: 1,377件（固定）

## 各ステップの手順
1. mixed JSONL作成: `cat data/independence.jsonl + head -N data/wo_independence.jsonl`
2. 学習コマンド:
```
pipenv run python mlp.py \
  --mode qa-train \
  --qa_path data/mixed_stepN.jsonl \
  --epochs 10 \
  --use_final_layer \
  -m "4d scaling step N: independence(1377) + wo(XXXX)"
```
3. 推論コマンド（λ=0.8）:
```
pipenv run python mlp.py \
  --mode infer \
  --model_dir model/YYYYMMDDHHMM_qwen25-05b-instruct-qa \
  --lambda_interp 0.8 \
  --prompt "原子力規制委員会の独立性について説明してください。"
```
4. 結果をこのファイルに追記（出力は省略せず全文記録）
5. Step 5まで実施

## 結果

### Step 1: independence only（1,377件）
- モデル: `model/202603121722_qwen25-05b-instruct-qa`
- 学習: loss E10=0.6610、4.5min
- 品質: ✓ 良質（3条委員会・両議院の同意・身分保障）

**λ=0.8 出力:**
原子力規制委員会は、日本の原子力安全規制に関する最高的な機関として知られています。この委員会の構成や機能については、原子力規制委員会設置法に基づき、以下のような特徴を示します：

1. 独立性：原子力規制委員会は、政治や経済政策、他の政府機関からの影響を受けないよう独立した「3条委員会」として設置されています。

2. 設定許可：原子力発電所の設置許可を受けることが可能となります。

3. 人事管理：委員長が任命され、任期中には任命権者の一存で罷免されることがあります。

4. 政策評価：専門家による評価を行い、科学的・専門技術的な判断をもとに規則を作成しています（国家行政組織法第36条第5項に基づく規定）（ただし、原子力規制委員会は、人格が高潔であることを確保するために、内閣の指揮下に設置されているため、その職権は国会同意人事とされており、身分保障の観点からも独立性が保たれていること）。

5. 原子力規制委員会設置法に基づく設置：原子力規制委員会は、独立性と中立性を確保するため、両議院の同意人事とされており（ただし、原子力規制委員会は、原子力発電所等規制に関しても、政治や経済政策、他の政府機関からの影響を受けないよう独立した「3条委員会」によって設置されています）。

このように、原子力規制委員会は、科学的・専門技術的な判断をもとに規則を作成し、核エネルギーの安全性を確保することを目指す重要な機関です。

原子力規制委員会の独立性は、国家行政機関が実効的に独立してその職権を行使するという意味であり、国家行政組織法に基づく「3条委員会」として設置されたことからも明らかです。
