# 4B fp32 Adam 5ep kangaekata 学習レポート

## 目的
8-bit AdamでNaN発散が発生した原因をfp32 Adamに戻して検証しつつ、kangaekata_rag_workflowデータセット（273,516件）でのQwen3-4B MLP Memory学習品質を評価する。

### モデルパス
model/202603182337_qwen3-4b-instruct-2507-qa

### ベースモデル
Qwen/Qwen3-4B-Instruct-2507（hidden_size=2560, num_hidden_layers=36）

### アーキテクチャ
- target_layer: 25（= int(36 * 0.7)）
- use_final_layer: true（h = layer25 + final layer）
- num_layers (M): 8（MLPBlock 残差スタック数）
- FFN hidden_dim: 2560 × 4 = 10240（4d）
- 総パラメータ: 約419.2M（52.4M × 8ブロック）

### データ
- 学習JSONL: data/202510301810_250304_kangaekata_rag_workflow.jsonl（273,516件）

### 学習設定
- mode: qa-train
- batch_size: 32, epochs: 5
- optimizer: AdamW fp32（lr=4e-4）
- max_tokens_per_step: 2048
- 損失: E1=0.9464 → E2=0.6504 → E3=0.5485 → E4=0.4758 → E5=0.4131
- 学習時間: 19.4h（1,163min）
- GPU: RTX 3090 24GB, CUDA 12.2
