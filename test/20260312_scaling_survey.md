# スケーリングサーベイ — 独立性クエリ品質 vs データ量

## 目的
independence.jsonl に wo_independence.jsonl（元JSOLから独立性関連を除いたノイズデータ）を
2倍ずつ増やしながら学習し、独立性クエリの品質がいつ劣化するかを調べる。

## データ準備
- 元JSONL: `data/202510301810_250304_kangaekata_rag_workflow.jsonl`（273,516件）
- `data/wo_independence.jsonl`: 元JSOLから「原子力規制委員会」AND「独立性」を含む行を除外（272,139件）
- `data/independence.jsonl`: 1,377件（固定）

## 各ステップの手順
1. mixed JSONL作成: `cat data/independence.jsonl + head -N data/wo_independence.jsonl`
2. 学習コマンド:
```
pipenv run python mlp.py \
  --mode qa-train \
  --qa_path data/mixed_stepN.jsonl \
  --epochs 10 \
  --use_final_layer \
  -m "scaling step N: independence(1377) + wo(XXXX)"
```
3. 推論コマンド（λ=0.8）:
```
pipenv run python mlp.py \
  --mode infer \
  --model_dir model/YYYYMMDDHHMM_qwen25-05b-instruct-qa \
  --lambda_interp 0.8 \
  --prompt "原子力規制委員会の独立性について説明してください。"
```
4. 結果をこのファイルに追記（出力は省略せず全文記録）
5. 品質が著しく低下したら終了
